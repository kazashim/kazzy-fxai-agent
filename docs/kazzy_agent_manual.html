4. Platform API Setup Guide

🔴 CRITICAL SECURITY: Never enable "Withdrawal" permissions on your API keys. Only enable permissions necessary for trading. Always restrict API access to specific IP addresses when possible.

4.1 Binance API Setup

Step-by-Step Instructions

  1. Log in to your Binance account at https://www.binance.com
  2. Navigate to Profile → API Management (or click your profile icon and select API)
  3. Click the "Create API" button
  4. Enter a unique label for your API key (e.g., "KazzyAgent")
  5. Complete security verification (2FA)
  6. Important: Copy your API Key and Secret Key immediately - the Secret Key will not be shown again
  7. Edit the API key permissions:
    • Enable Reading permission
    • Enable Trading permission (required for auto-trading)
    • Disable Withdrawals permission
  8. (Optional) Set IP restrictions for enhanced security
ℹ️ Note: Binance offers testnet for testing. Use testnet keys for initial setup and validation before connecting live accounts.

4.2 MetaTrader 5 (MT5) API Setup

Option A: MetaQuotes Cloud (Recommended)

  1. Download and install MT5 terminal from https://www.metatrader5.com/
  2. Create or log into your MT5 account
  3. Go to Tools → Options → API
  4. Enable API access checkbox
  5. Note the server address and login credentials
  6. Use these credentials to connect through Kazzy Agent

Option B: Third-Party REST API

  1. Register at https://metatraderapi.cloud/
  2. Follow their setup guide to obtain API credentials
  3. Use the provided credentials in Kazzy Agent

4.3 Coinbase API Setup

Step-by-Step Instructions

  1. Log in to Coinbase at https://www.coinbase.com
  2. Go to Settings → API Access
  3. Click "+ New API Key"
  4. Configure permissions:
    • ✅ View permission - Required
    • ✅ Trade permission - Required for trading
    • ❌ Transfer permission - DO NOT ENABLE
  5. Add your IP address to the whitelist (if required)
  6. Save your API Key and Secret

4.4 Bybit API Setup

Step-by-Step Instructions

  1. Log in to Bybit at https://www.bybit.com
  2. Click your profile icon in the top-right corner
  3. Select API from the menu
  4. Click "Create New Key"
  5. Enter a name for your API key
  6. Select permissions:
    • ✅ Read permission
    • ✅ Trade permission
    • ❌ Withdraw permission - DO NOT ENABLE
  7. Complete security verification (Google Authenticator)
  8. Copy your API Key and Secret immediately

4.5 Interactive Brokers (IBKR) API Setup

Step-by-Step Instructions

  1. Log in to IBKR Client Portal at https://portal.interactivebrokers.com
  2. Navigate to Settings → API Settings
  3. Click "Create API Key"
  4. Configure permissions:
    • Read market data
    • Place orders
    • View positions
  5. Set IP restrictions (recommended)
  6. Download and save your credentials
ℹ️ Note: IBKR requires the Trader Workstation (TWS) to be running for API connections. Make sure TWS is installed and logged in.

4.6 Kraken API Setup

Step-by-Step Instructions

  1. Log in to Kraken at https://www.kraken.com
  2. Go to Security → API
  3. Click "Generate New Key"
  4. Set key permissions:
    • ✅ Query funds/ledger/trade
    • ✅ Query open orders/trades
    • ✅ Create and close orders
    • ❌ Withdrawal funds - DO NOT ENABLE
  5. Add IP whitelist if desired
  6. Copy both the API Key and Private Key

4.7 Connecting to Kazzy Agent

After obtaining your API keys from any exchange:

  1. Open the Kazzy Agent dashboard
  2. Navigate to the Settings tab
  3. Select your trading platform from the list
  4. Enter your API Key and API Secret
  5. Toggle "Testnet" for testing before live trading
  6. Click "Connect Exchange"
  7. Verify connection status shows "Connected"

6. Auto-Trading Configuration

6.1 Enabling Auto-Trading

Kazzy Agent can automatically execute trades based on AI-generated signals. Here's how to enable it:

  1. Connect at least one exchange with valid API keys
  2. Navigate to the Automation tab in the dashboard
  3. Enable the "Master Control" toggle
  4. Configure your risk parameters:
    • Max Risk per Trade: Recommended 1-2%
    • Max Daily Loss: Recommended 5-10%
    • Maximum Open Positions: Based on your risk tolerance
  5. Select trading strategies:
    • RSI Mean Reversion: Buy oversold, sell overbought
    • MA Crossover: Trend-following strategy
    • Grid Trading: Range-bound markets
  6. Click "Start Auto-Trading"
⚠️ Risk Warning: Always configure risk parameters before enabling auto-trading. Start with conservative settings and adjust based on performance.

6.2 AI Signal Generation

Kazzy Agent analyzes markets using multiple technical indicators:

  • RSI (Relative Strength Index): Identifies overbought/oversold conditions
  • MACD: Trend momentum and potential reversals
  • Moving Averages: Support and resistance levels
  • Volume Analysis: Confirms price movements
  • Pattern Recognition: Identifies chart patterns

Each signal includes a confidence score (0-100%) to help you assess trade quality.

9. Security Best Practices

🔴 SECURITY IS Paramount: Failure to follow these guidelines may result in loss of funds. Always prioritize security when configuring automated trading systems.

9.1 API Key Security

  • Never share your API keys with anyone
  • Enable IP restrictions on API keys when possible
  • Use Read-Only permissions if you only need data
  • Enable Trade permissions only when actively using auto-trading
  • Regularly rotate your API keys (every 30-90 days)
  • Never store API keys in plain text files
  • Use environment variables or secure password managers

9.2 Exchange Account Protection

  • Enable Two-Factor Authentication (2FA) on all exchanges
  • Use unique, strong passwords
  • Monitor your accounts regularly
  • Set up withdrawal address whitelisting
  • Review API permissions monthly
  • Enable login alerts for account access

9.3 Trading Risk Controls

Always configure these before enabling auto-trading:

  • Maximum risk per trade: 1-2% of account balance
  • Daily loss limit: 5-10% of account balance
  • Maximum concurrent positions: Based on risk tolerance
  • Emergency stop button: Ensure it's always accessible
⚠️ Emergency Stop: Familiarize yourself with the emergency stop function. In case of unexpected behavior, immediately stop all trading to prevent further losses.

9.4 Server Security

  • Run Kazzy Agent on a VPS for 24/7 operation
  • Use firewall rules to restrict access
  • Enable SSH key authentication (disable password login)
  • Keep all software up to date
  • Use VPN for remote access
  • Enable SSL/TLS for all connections

10. Troubleshooting

10.1 Common Connection Issues

Problem: Cannot connect to exchange

  • Verify API key and secret are correct (no extra spaces)
  • Check if IP is whitelisted on exchange
  • Ensure API has trading permissions enabled
  • Verify account is verified on the exchange
  • Check if exchange API is experiencing outages

Problem: API key invalid error

  • Regenerate API keys on exchange
  • Check for extra spaces in key fields
  • Verify key hasn't expired
  • Ensure correct API key type (spot vs futures)

10.2 Trading Issues

Problem: Orders not executing

  • Check account has sufficient balance
  • Verify symbol is available on exchange
  • Check if market is open for the symbol
  • Review risk limits haven't been reached
  • Check minimum order size requirements

Problem: Position not closing

  • Use Emergency Stop for immediate closure
  • Check internet connection
  • Verify API has 'Trade' permissions
  • Check if position already closed by exchange

10.3 Performance Issues

Problem: Slow data updates

  • Check internet connection speed
  • Disable unnecessary browser tabs/extensions
  • Use wired connection instead of WiFi
  • Close other bandwidth-intensive applications
  • Consider upgrading your server/VPS

Problem: AI not generating signals

  • Ensure Poe API key is configured in Settings
  • Check AI model is selected
  • Verify exchange is connected with data access
  • Check API rate limits
  • Review error logs in Terminal panel

10.4 Getting Help

If you encounter issues not covered in this guide:

  • Check Kazzy Agent documentation online
  • Review exchange API documentation
  • Contact exchange support for API-specific issues
  • Check system logs for detailed error messages
